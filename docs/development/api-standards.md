# API 设计规范

## RESTful API 规范
### 请求方法
- GET：查询
- POST：创建
- PATCH：部分更新
- PUT：完全更新
- DELETE：删除

### URL 命名
- 使用复数名词
- 使用小写字母
- 使用连字符（-）连接单词
- 避免使用动词

### 状态码使用
- 200：成功
- 201：创建成功
- 400：请求错误
- 401：未认证
- 403：权限不足
- 404：资源不存在
- 500：服务器错误

## 响应格式
```json
{
  "status": "success|error",
  "data": {},
  "message": "错误信息（仅在发生错误时返回）"
}
```
